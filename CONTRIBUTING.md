# Contributing to AI-Powered Release Notes Generator

Thank you for your interest in contributing! Here's how to get started.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/<your-username>/generate-release-notes.git`
3. Create a branch: `git checkout -b feature/your-feature`
4. Install dependencies: `npm run install:all`
5. Make your changes
6. Test your changes locally with `npm run dev`

## Development Setup

See [README.md](README.md#-installation) for full setup instructions.

## Pull Request Process

1. Ensure your code works locally (both frontend and backend)
2. Update documentation if you've changed APIs or added features
3. Use [conventional commits](https://www.conventionalcommits.org/) for commit messages:
   - `feat: add new feature`
   - `fix: resolve bug`
   - `docs: update documentation`
4. Submit a PR against the `main` branch

## Reporting Issues

- Use GitHub Issues to report bugs
- Include steps to reproduce, expected vs actual behavior, and your environment (OS, Node.js version)

## Code Style

- TypeScript for both frontend and backend
- Follow existing patterns in the codebase
- Keep PRs focused — one feature or fix per PR
